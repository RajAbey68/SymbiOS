# SymbiOS Mandatory 4-Eyes Standard (Bootstrap v1)

This document outlines the mandatory **4-Eyes Principle** (Maker-Checker) requirement for all project bootstraps within the **SymbiOS** ecosystem.

## 1. Requirement Summary
All automated high-value or high-risk actions generated by AI agents MUST be subject to human oversight before final execution. This is a primary requirement for **EU AI Act compliance**.

## 2. Mandatory Implementation
Every project bootstrap MUST include the following components:

### A. Governance Initialization (`bootstrap.ts`)
A dedicated startup script that verifies the presence of the `ActionIntentQueue` and ensures the database is ready for governance logging.

### B. Route Protection (`FourEyesMiddleware.ts`)
A standard middleware that intercepts requests to sensitive endpoints and ensures they are metadata-tagged with a `Maker Identity`.

### C. HIL Gateway (`HilGatewayService.ts`)
A centralized service that evaluates an `ActionIntent` against the 4-Eyes policy (Confidence score, Dollar limits, and Maker-Checker segregation).

## 3. Compliance Traceability
Every pipeline execution trace MUST include the following audit metadata:
- `fourEyesRequirementMet`: Boolean indicating the 4-eyes status.
- `makerIdentity`: The identity of the submitting agent.
- `checkerIdentity`: The identity of the reviewing human.

## 4. Strict Mode
In production environments, the server MUST fail to boot if the 4-Eyes Governance layer cannot be initialized.
